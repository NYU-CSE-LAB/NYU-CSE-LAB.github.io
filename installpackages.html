How to compile Linux Binary on your local account?

by Amit Biswas Posted Feb. 16, 2017

Before you start, make sure you know how to SSH to the server. Learn about that here.

If you are home (read: not connected to NYU wifi) you have to use NYU VPN). Scroll down the page to see the download link for the vpn client for your platform (Windows, Mac etc.). When the vpn client asks for the server address use vpn.nyu.edu and use your typical netid and password to authenticate.

There are instances where you require sudo access to install different packages on your lcoal machine. There's a way to install those packages without sudo access. We install those packages on our local machine and modify the path in our home directory. This way the shell would look up in installed binary directory before it checks the system directories.

Here's an example which explains it better.
  1. downloaded the cmake 3.8.0 binary from : wget https://cmake.org/files/v3.8/cmake-3.8.0-rc2-Linux-x86_64.sh
  2. Then add execute permission : chmod +x cmake-3.8.0-rc2-Linux-x86_64.sh
  3. Now run it with the command : bash cmake-3.8.0-rc2-Linux-x86_64.sh
  4. It will give you the license agreement, hit space bar to scroll down, then accept the license by entering y. When you accept, it will tell you the installation directory.

    5. In my case it shows the following: By default the CMake will be installed in: "/home/abiswas/cmake-3.8.0-rc2-Linux-x86_64" Do you want to include the subdirectory cmake-3.8.0-rc2-Linux-x86_64? Saying no will install in: "/home/abiswas" [Yn]:

    Enter 'Y' and it will start extracting.

    There should be a new directory now, in your home directory. Simply run the cmake from there:

    If you are in your home directory run the following: ./cmake-3.8.0-rc2-Linux-x86_64/bin/cmake --version #./ refers to the current directory

    By default, you bash shell will use the cmake installed in the system. See below: [abiswas@code ~]$ cmake --version cmake version 2.8.12.2

  5. Execute : ls -la
  6. Exacute : vim .bashrc or nano .bashrc (not .bash_rc)
  7. Exacute : vim .bashrc or nano .bashrc (not .bash_rc)
  8. Append following line in .bashrc file : export PATH=$HOME/cmake-3.8.0-rc2-Linux-x86_64/bin:$PATH

    9. I am basically telling the shell to look inside my cmake directory before looking at system directories to find cmake.

  9. Now source the file (let the shell update its environment) : source .bashrc
  10. To test our changes execute : cmake --version or which cmake

    11. Now it is using the cmake I installed in my home directory.

    Hope this makes everything clear. You can install almost every linux binary package this way.
